Math.Units

Pixels Module

Namespace: Math.Units
Assembly: Math.Units.dll

Although most of the focus of elm-units is on physical/scientific units, it's often useful to be able to safely convert back and forth between (for example) [Length](Length) values in the real world and on-screen lengths in pixels. This module provides a standard Pixels units type and basic functions for constructing/converting values of type Quantity Int Pixels or Quantity Float Pixels, which allows you to do things like represent conversions between real-world and on-screen lengths as [rates of change][1]. This in turn means that all the normal [Quantity](Quantity) functions can be used to convert between pixels and other units, or even do type-safe math directly on pixel values. [1]: Quantity#working-with-rates
